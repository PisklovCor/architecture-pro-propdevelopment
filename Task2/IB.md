## Проверочный лист безопасности для бизнес-систем PropDevelopment

### Обоснование выбора разделов

На основе анализа архитектуры и проблем компании PropDevelopment были выбраны следующие разделы:

**I. Управление доступом (КРИТИЧНЫЙ ПРИОРИТЕТ)**
- **Обоснование:** Основная выявленная проблема - клиенты видят данные других клиентов в личном кабинете. Это прямое нарушение принципа минимальных привилегий и разграничения доступа. Несколько точек регистрации клиентов без координации усугубляют проблему. Отсутствует централизованное управление доступом (IAM/IDM).

**II. Безопасность данных (КРИТИЧНЫЙ ПРИОРИТЕТ)**
- **Обоснование:** Нарушен контроль данных - это одна из двух глобальных проблем компании. Внутренние потоки данных между IT-продуктами не контролируются системно. API партнёров передают персональные данные без ограничений. Проблема блокирует развитие направления BI/ML/AI. Компания обязана соблюдать 152-ФЗ о персональных данных.

**III. Защита инфраструктуры (ВЫСОКИЙ ПРИОРИТЕТ)**
- **Обоснование:** Гибридная инфраструктура (собственный ЦОД + облако провайдера) требует комплексного подхода к защите. Команды используют облачные ресурсы для доработки/тестирования без единых стандартов безопасности. Необходима сегментация сети и контроль периметра.

**IV. Управление инцидентами (КРИТИЧНЫЙ ПРИОРИТЕТ)**
- **Обоснование:** У конкурента произошла утечка данных, руководство обеспокоено. Один специалист по ИБ на всю компанию не может обеспечить мониторинг и своевременное реагирование на инциденты. Текущие инциденты (отображение чужих данных) не обнаруживаются вовремя. Отсутствует SIEM и автоматизация обнаружения угроз.

**V. Специфические аспекты (ВЫСОКИЙ ПРИОРИТЕТ)**
- **Обоснование:**
  - Мобильное приложение для собственников (оплата ЖКХ, управление Умным домом) требует MDM
  - API партнёров (управляющих компаний) - критическая точка уязвимости с избыточными правами доступа
  - Интеграция с госорганами (онлайн-сделки) требует повышенного уровня защиты
